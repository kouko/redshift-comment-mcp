# redshift-explore

[English](README.md) · **日本語** · [繁體中文](README.zh-TW.md)

## 何をするスキル？

`redshift-explore` は、初めて触る Redshift クラスタを案内付きで歩くた
めのスキルです。**スキーマ → テーブル → 列** の 3 ステップ・ウィザー
ドで、各ステップで候補をコメント付きで一覧表示します。ポイントは、
**名前を覚えるのではなく、コメントを読んで選ぶ** こと。`fct_orders`
が `dbt_marts` 配下にある、なんて記憶している必要はありません。リスト
を上から読んで、目的に合うコメントを見つけ、番号で答えるだけです。

## こんなときに使う

新しいクラスタに初日から触れるとき、引き継いだウェアハウスを開いたと
き、あるいは「**どこから見ればいいか分からない**」と思ったときに使い
ます。**ゼロコンテキスト・オンボーディング** が前提の設計です。すで
にテーブル名と列名が分かっているなら、ウィザードを飛ばして
`/redshift-profile` を直接呼び出してください。

## 対話例

```
> /redshift-explore

スキーマを選んでください:
  1. dbt_marts     — マート層、業務向けファクト・ディメンション
  2. dbt_staging   — ステージング、raw を軽く整形
  3. raw_orders    — orders サービスのイベント生データ
返信: 番号 / 名前 / キーワード。

> 売上

「売上」で絞り込み:
  1. dbt_marts     — 売上集計を含む最終マート層
返信: 番号 / 名前 / back。

> 1

dbt_marts のテーブルを選んでください:
  1. fct_orders     — 完了注文 1 件 = 1 行、粒度 = order_id
  2. fct_revenue    — 月次売上サマリ
返信: 番号 / 名前 / キーワード / back。
```

列を選ぶと、`/redshift-profile` に引き継がれて、値分布・NULL 率・トッ
プ N を確認できます。

## 抜け道（どのステップでも有効）

- **直接識別子** — `dbt_marts.fct_orders` と入力すれば一気に進みます。
- **キーワード** — `revenue` と打てば、検索で絞り込み再表示。
- **`back`** — 一つ前のステップへ。
- **`cancel`** / **`quit`** — 何も残さず終了。

詳細仕様は [SKILL.md](./SKILL.md) を参照。
